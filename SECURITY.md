# Security Policy

## Reporting a Vulnerability

The OriginTrail team takes security seriously. If you discover a security vulnerability in DKG V9, we appreciate your help in disclosing it responsibly.

**Please do NOT open a public GitHub issue for security vulnerabilities.**

Instead, report vulnerabilities by emailing:

**security@origin-trail.com**

Please include:

- A description of the vulnerability.
- Steps to reproduce the issue.
- Any potential impact you've identified.
- Suggested fix, if you have one.

## Response Timeline

- **Acknowledgment**: We will acknowledge your report within 48 hours.
- **Assessment**: We aim to assess the severity within 5 business days.
- **Fix**: Critical vulnerabilities will be patched as quickly as possible, with a public advisory once the fix is deployed.

## Scope

This policy applies to the code in this repository and the DKG V9 protocol implementation. For smart contract vulnerabilities in `@origintrail-official/dkg-evm-module`, the same process applies.

## Supported Versions

| Version | Supported |
|---|---|
| 9.x (latest) | Yes |
| < 9.0 | No |

## Recognition

We are grateful to security researchers who help keep the DKG ecosystem safe. With your permission, we will credit you in the release notes for any vulnerability you report.
